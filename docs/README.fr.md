# <a id="philosophy">🌱&nbsp;</a> Quel projet ?

J'ai souvent envie de partir à l'aventure en codant tout de suite quitte à ne pas être rigoureux. Malheureusement, on récolte toujours ce que l'on sème !

Ce projet a donc pour but de ne pas trop tuer la créativité d'un projet au début en suivant les meilleures pratiques de code ou en phase avancé nécessitant une réécriture de tas de lignes de code.
This template is too avoid that big bill at the end so remember **coder ces idées comme un fou mais accompagné et de manière standardisé pour être facilement prêt à être lancé en production** 🚀

# <a id="usage">📖&nbsp;</a> Utilisation

<br/>

## 🚀&nbsp; Rapide démarrage

Assez simple à utiliser !

Lance ces commandes en local:

```shell
git clone --depth=1 https://github.com/jonitoh/ts-project-template.git my-project-name
cd my-project-name
npm install
```
Tu peux créer le fichier `.env` et y ajouter ceci:

```shell
NODE_ENV=development
```

Enfin, lance cette ligne de commande:

```shell
npm run dev
```

Voilà !!!

## ⚙️&nbsp; Spécifications nécessaires

Un environnement où utiliser Node.js

## 👀&nbsp; Exemples

Après avoir lancé la commande `npm run dev` ou encore `npm run start`, un message similaire devrait apparaitre sur le terminal:
```
YYYY-MM-DD HH:mm:ss [dist/index.js] INFO :: This is the name you chose: Maximilian The Third

YYYY-MM-DD HH:mm:ss [dist/index.js] INFO :: Its uppercase version: --MAXIMILIAN THE THIRD--

YYYY-MM-DD HH:mm:ss [dist/index.js] INFO :: Its length: 24
```

# <a id="features">🎯&nbsp;</a> Fonctionnalités (en anglais)

-   🔥 Type checking [TypeScript](https://www.typescriptlang.org)
-   ✅ Strict Mode for TypeScript
-   📏 Linter with [ESLint](https://eslint.org)
-   💖 Code Formatter with [Prettier](https://prettier.io)
-   🦊 [Husky](https://typicode.github.io/husky/#/) for Git Hooks
-   🚫 [Lint-staged](https://github.com/okonet/lint-staged) for running linters on Git staged files
-   🚓 Lint git commit with [Commitlint](https://commitlint.js.org/#/)
-   📓 Write standard compliant commit messages with [Commitizen](https://commitizen-tools.github.io/commitizen/)
-   🎁 Add emojis in commit messages
-   🦺 Unit Testing with [Jest](https://jestjs.io/) and coverage
-   🗂 VSCode configuration: Settings, ESLint, Prettier
-   👷 A decent `.github` folder
-   🧪 custom logging with [Winston](https://github.com/winstonjs/winston)
-   🌈 multilingual README
-   🔍 consistent coding styles across editors with [editorconfig](https://editorconfig.org/)
-   ⚙️ environment variable management with `cross-env` and `dotenv`
-   ⚡ strict versionning using `.npmrc` and `.nvmrc`

# <a id="limits">⚠️&nbsp;</a> Limitations (en anglais)

I try to stay the most generic possible so that it can be used for many purpose.

Below is more like a list of things to do:
-   🧪 E2E Testing with Cypress
-   👷 Run tests on pull request with GitHub Actions
-   🎁 Automatic changelog generation with Semantic Release
-   💡 Absolute Imports using `@` prefix
-   🗂 VSCode configuration: Debug, Settings, Tasks and extension for TypeScript, Jest
-   🖱️ Add test to custom logger (help [here](https://www.youtube.com/watch?v=BfA7MosIgik), [here](https://stackoverflow.com/questions/57771616/) and [here](how-to-use-jest-to-mock-winston-logger-instance-encapsulated-in-service-class)) and [colorizing](
